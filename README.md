# KeChat 可聊

> 💬 **轻量、自主可控的企业内部即时通讯 + AI 驻守助理**
>
> 一套聊天框架，一个 AI 驻守，让数据不出自家服务器，让业务推进不再靠人追人。

---

## 🎯 解决的问题

### 为什么小团队需要我们

- **用大平台？** 微信/钉钉/飞书配置复杂，企业版收费不菲，数据在别人手里
- **自己搭？** 技术门槛高，缺乏统一框架，AI 集成更是无从下手
- **最根本的问题：** 公司需要一个**驻守的 AI 助理**——它能时刻在线，跟踪资源、盯进度、跟业务人员勾兑任务完成情况、提醒日程——而不是一家家去对接大模型的付费 API

**KeChat 的答案：**
> 开源、自托管 IM 框架 + AI Agent 驻守，前端你聊，后端 AI 帮你盯。小公司也能拥有自己的"数字员工"。

### 适用场景

- 不想让内部沟通数据经过第三方的小团队
- 需要 AI 助理主动盯任务、催进度、调资源的弹性团队
- 希望通过开源自建降低长期运营成本的创业公司
- 为申请大模型 Token 资源（如小米 MiMo）提供真实项目支撑的开发者

## 🚀 快速开始

### 方式一：Docker 运行（推荐）

```bash
# 克隆仓库
git clone https://github.com/lilly7998/kechat.git
cd kechat

# 启动服务
docker compose up -d

# 访问 http://localhost:8088
```

### 方式二：直接运行

```bash
cd backend
pip install -r requirements.txt
python main.py
```

## 📡 API 文档

启动服务后访问 `http://localhost:8088` 可查看聊天界面。

### 核心接口

| 方法 | 路径 | 说明 |
|:---|:----|:----|
| POST | `/api/register` | 注册新用户 |
| POST | `/api/login` | 登录获取 token |
| GET | `/api/users?token=xxx` | 获取用户列表 |
| GET | `/api/messages/{peer_id}?token=xxx` | 获取聊天历史 |
| GET | `/api/conversations?token=xxx` | 获取会话列表 |
| WS | `/ws?token=xxx` | WebSocket 实时聊天 |

## 🏗 技术栈

- **后端**: Python FastAPI + WebSocket
- **数据库**: SQLite（轻量，零配置）
- **部署**: Docker
- **前端**: 纯 HTML/CSS/JS（无需构建工具）
- **AI 引擎**: 接入大模型 API（小米 MiMo / OpenAI / 智谱 等）

## 🤖 AI 驻守助理（规划中）

### 设计理念

> 团队聊天的"第四人"，不说话的时候也在盯着。

传统的 IM 只是消息管道，而 KeChat 的 AI 助理是一个**常驻群聊/私聊的系统级 Agent**，它像公司的一个真实员工那样被@、被安排、被问责。

### 核心能力

| 能力 | 说明 |
|:----|:-----|
| 🎯 **任务追踪** | 自动识别聊天中的任务分配，生成待办清单，跟踪完成状态。@AI 问"小王那个方案改完了吗"——它立刻调取上下文给出进度 |
| 📊 **资源调度** | 对接服务器、GPU、云资源 API，实时掌握资源水位。超阈值自动在群内预警 |
| 📅 **日程提醒** | 根据聊天中约定的时间节点主动提醒，预判延期的任务自动@相关负责人 |
| 💬 **业务勾兑** | 业务人员可以像聊天一样直接问 AI ——"这个项目的测试环境什么时候就绪？""昨天部署的版本有没有报错？" |
| 📋 **周报生成** | 自动汇总一周内完成的任务、未完成的卡点、资源使用趋势，生成摘要发到群里 |
| 🔔 **主动推送** | 不等人问，AI 发现异常（服务挂了、资源告急、任务延期）会主动在群里发声 |

### 为什么需要大模型 Token

AI 助理不是玩具——它需要持续的推理能力来处理业务对话、理解上下文、做出判断：

- **开发阶段**：调试 Agent 逻辑、优化 Prompt、测试工具调用
- **运维阶段**：7×24 在线驻守，持续消耗推理 Token
- **额度估算**：一个日均活跃 10 人、100 次对话的小团队，日消耗约 3–5M Token

**这就是为什么我们申请小米 MiMo 100 万亿 Token 免费计划**——让 AI 驻守从成本上对小公司真正可行。没有免费额度，一个自建 Agent 每月 API 费用就可能超过服务器本身。MiMo 开源 + 赠额的组合，让"小公司的数字员工"不再是噱头。
